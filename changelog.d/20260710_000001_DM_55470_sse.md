### New features

- The `GET /v1/pages/{page}/html/events` SSE stream now reaches a terminal execution-failure state. When a page instance's Noteburst job completes as an execution failure (no executed notebook), the event payload carries a new optional `execution_error` object with a machine-readable `code` and a human-readable `title` and `message`, so a subscribed client can stop and render the failure. The stream performs the same cleanup as the interactive `htmlstatus` path (it writes the failure marker and deletes the stale Noteburst job record). This field is additive and backward compatible: it is null in the normal, pending, and available cases.
