### New features

- The `GET /v1/pages/{page}/htmlstatus` endpoint now returns a terminal execution-failure result instead of HTTP 500 when Noteburst reports that a notebook could not be executed. The `HtmlStatus` response gains a new optional `execution_error` object carrying a machine-readable `code` and a human-readable `title` and `message`. A cached failure short-circuits re-execution so a persistently broken notebook no longer triggers a fresh Noteburst execution on every poll. This field is additive and backward compatible: it is null in the normal, pending, and available cases.
