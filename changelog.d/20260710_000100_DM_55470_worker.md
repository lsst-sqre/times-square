### Bug fixes

- The scheduled and background worker tasks (`scheduled_page_run` and `replace_nbhtml`) now log a structured warning and continue when Noteburst reports that a notebook could not be executed, instead of raising and posting a generic "Times Square worker exception" message to Slack. An infrastructure-level execution failure (timeout, Jupyter error, unknown, or an expired result) is an expected terminal outcome, not a bug, so it no longer spams the alerting channel. The stale Noteburst job-store record is cleaned up on failure. A genuine contract violation (Noteburst reports `success=true` but returns no notebook) still raises loudly.
