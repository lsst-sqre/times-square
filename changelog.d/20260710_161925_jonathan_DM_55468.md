### Bug fixes

- Fixed the pull-request notebook-execution check so that per-notebook timeout annotations report the job's real runtime and state. The Noteburst status lookup in the timeout-reporting path was never awaited, so its result was silently dropped and every timeout annotation showed a bare message with no runtime. The call is now awaited, so an annotation for a timed-out job reports how long the job had been running and whether it was still in progress or still queued. A failed or missing Noteburst status lookup still reports the timeout gracefully with no runtime.
